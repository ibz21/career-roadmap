\# Week 1 - Git Basics



\## What is Git?

Git is a version control system.



That means it tracks changes to files over time and lets me save versions of a project in a structured way.



\## Why is Git used?

Git is used so that changes can be:

\- tracked

\- reviewed

\- saved as checkpoints

\- undone if needed

\- shared with remote platforms like GitHub



\## What is GitHub?

GitHub is an online platform that hosts Git repositories.



Git works locally on my computer.  

GitHub is the remote online copy.



\## Local Git vs GitHub

\- \*\*Git\*\* = local version control on my computer

\- \*\*GitHub\*\* = remote online copy of the repository



\## What is a repository?

A repository is a project folder that Git is tracking.



When I ran `git init`, my normal folder became a Git repository.



\## What is the working directory?

The working directory is the current version of my files on my computer that I am editing.



This is where I make normal changes to notes, labs, and project files.



\## What is the staging area?

The staging area is Git’s preparation area.



When I use `git add`, I am telling Git which changes should go into the next commit.



\## What is a commit?

A commit is a saved checkpoint in Git history.



It is a named version of the project at a certain point in time.



\## What is a branch?

A branch is a line of work in the repository.



At the moment, I am working on the `master` branch, which is the main line of work for this repo.



\## What is a remote?

A remote is a connected online copy of the repository.



In this project, the remote is the GitHub repo linked with:



`origin`



\## What does push mean?

Push means sending local commits to the remote GitHub repository.



This is the step that updates the online copy.



\## Simple Git flow

\- edit files locally

\- stage changes with `git add`

\- save a checkpoint with `git commit`

\- upload to GitHub with `git push`



\## Simple mental model

\- save file = save on computer

\- `git add` = prepare changes for commit

\- `git commit` = save checkpoint in Git history

\- `git push` = upload checkpoint to GitHub



\## Why this matters

Git and GitHub are core tools for:

\- software development

\- collaboration

\- version history

\- portfolio visibility

\- safe project progress
